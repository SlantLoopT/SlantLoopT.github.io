const,引用与指针

对const的引用（常量引用）

引用常量必须使用对const的引用，但所引用对象可以是常量也可以是变量。不能用常量引用初始化一个非常量引用。

此引用(b)不能修改所引用对象的值,无论所引用对象是常量还是变量。

在 C++ 中，const T& 是唯一可以绑定到常量对象、右值、临时值（表达式结果）的“左值引用类型”。

情况 int& 是否能绑定 const int& 是否能绑定
int 左值 ✅ 是 ✅ 是
const int 左值 ❌ 否 ✅ 是
字面值 42 ❌ 否 ✅ 是
表达式 a + b ❌ 否 ✅ 是（临时变量）

由于引用(b)不是一个对象，所以不存在引用(b)是常量还是变量，也就是说常量引用这个说法其实是不正确的。

常量指针（指向常量的指针）

const 数据类型 *变量名

  • 指向常量必须使用指向常量的指针，但所指向的对象可以是常量也可以是变量。
  • 不能通过解引用的方式修改所指向的常量或变量的值（若指向的是变量，则可以通过变量名修改）。
  • 不能用指向常量的指针初始化一个非指向常量的指针。
  • 注意事项
    • 指向的对象可以改变。
    • 一般用于修饰函数的形参，表示不希望在函数中修改内存地址中的值。
    • 如果用于形参，虽然指向的对象可以改变，但没有意义。
    • 如果形参的值不需要改变，建议加上const修饰。

此指针(b)指向常量(a),也就是它的值为常量的地址，且此指针(b)不能修改所指向对象的值,但此指针可以修改指向其他对象。

指针常量（const指针）与常指针常量

指针常量：数据类型 *const 变量名
常指针常量：const 数据类型 *const 变量名
常量指针必须初始化，一旦初始化完成，它的值（指向对象的地址）就不能改变了。变量，常量都可指向。

  • 注意事项
    • 指向的对象不能改变。
    • 定义的同时必须初始化，否则没有意义。
    • 可以通过解引用修改内存地址的值。

常指针常量指向的对象不可以改变，不能通过解引用的方法修改内存地址的值。

指针常量能否修改所指向对象的值，取决于所指对象的类型。
若所指对象是常量，则不论是所指对象还是指针本身的值都无法修改。

若所指对象是变量，则可以通过此指针修改该值，但指针本身的值不能改变。

顶层const与底层const

用顶层const表示自己本身是个常量，底层const表示所指对象是一个常量.

执行对象的赋值操作

对于含有顶层const的，由于顶层const表示自己本身是个常量，所以无法作为左值，作为右值时，忽略顶层const，只看底层const：
赋值双方有相同的底层const资格，则可赋值。
赋值双方无相同底层const资格，const可以指向或绑定在非常量上，const不能初始化或赋值给非常量。
